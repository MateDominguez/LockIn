Create `src/lockin/graph/orchestrator.py` with the following structure:

**Module docstring:**
```
Multi-ticker orchestration wrapper.

Runs Macro Oracle once (shared market context), fans out per-ticker pipelines
concurrently via asyncio.Semaphore, then aggregates via a portfolio-level optimizer.

Architecture (D-01/D-02/D-03):
  1. Macro Oracle — runs ONCE, result injected into all per-ticker initial states
  2. asyncio.gather fan-out — each ticker runs the full 7-agent pipeline concurrently
     with Semaphore(max_concurrent) controlling API rate limits (D-04)
  3. Portfolio Optimizer — runs ONCE over all valid per-ticker results

Rate limiting (D-04):
  - asyncio.Semaphore(max_concurrent=2) — controls concurrent LLM calls
  - tenacity @retry on 429 errors with exponential backoff
  - asyncio.to_thread() — graph.invoke() is sync; run in thread pool
```

**Function 1: `_is_rate_limit_error(exc: Exception) -> bool`**
Check if exception message contains "429", "quota", or "rate" (case-insensitive). Returns bool.

**Function 2: `_invoke_graph_with_retry(graph, state, config) -> dict`**
Decorated with `@retry(retry=retry_if_exception(_is_rate_limit_error), wait=wait_exponential(multiplier=1, min=4, max=60), stop=stop_after_attempt(5))`. Calls `graph.invoke(state, config)` and returns result.

**Function 3: `_run_macro_oracle_standalone(reference_ticker: str) -> dict`**

**Function 4: `analyze_portfolio(tickers: list[str], max_concurrent: int = 2) -> dict[str, Any]`**
This is the main async entry point. Steps:

Step 1: Call `_run_macro_oracle_standalone(tickers[0])` to get shared macro context. This is synchronous (no await needed).

Step 2: Define inner `async def run_ticker(ticker: str) -> dict | Exception:` that:
- Acquires semaphore (`async with semaphore:`) BEFORE calling to_thread (critical ordering per Pitfall 5)
- Creates a fresh graph instance per ticker: `create_graph(checkpointer=MemorySaver())` (per Research Q1: one graph per ticker for thread safety)
- Creates initial state: `create_initial_state(ticker)`
- Updates state with macro_context dict (inject shared macro)
- Calls `await asyncio.to_thread(_invoke_graph_with_retry, graph, state, config)` with config `{"configurable": {"thread_id": f"portfolio-{ticker}"}}`
- Wraps in try/except: on exception, logs error and returns the exception object

Step 3: `ticker_results = await asyncio.gather(*[run_ticker(t) for t in tickers], return_exceptions=True)`

Step 4: Filter results:
```python
valid_results = [r for r in ticker_results if not isinstance(r, (Exception, BaseException))]
failed = [(t, r) for t, r in zip(tickers, ticker_results) if isinstance(r, (Exception, BaseException))]
```
Log warnings for failures.

Step 5: Call `portfolio_optimize(valid_results, tickers)` and return result.

**Function 5: `portfolio_optimize(ticker_states: list[dict], all_tickers: list[str]) -> dict[str, Any]`**
Portfolio-level aggregation (D-03). Import constants inside function:
```python
from lockin.agents.optimizer import MAX_POSITION_SIZE, MAX_SECTOR_ALLOCATION
```

Logic:
1. Iterate valid ticker_states. For each, extract `asset_ticker`, `optimizer_portfolio`, `optimizer_sectors`, and the full state as `ticker_detail`.
2. Build `positions: dict[str, float]` mapping ticker -> position_size (capped at MAX_POSITION_SIZE).
3. Build `sector_allocations: dict[str, float]` accumulating sector weights across tickers.
4. Enforce cross-ticker sector cap: if any sector > MAX_SECTOR_ALLOCATION, scale down ALL positions in that sector proportionally: `scale = MAX_SECTOR_ALLOCATION / total_sector_weight`. Find which tickers belong to the over-allocated sector by checking their `optimizer_sectors`. Scale their position_size by `scale`.
5. Compute `total_allocation = sum(positions.values())`.
6. Collect `ticker_details: list[dict]` with per-ticker summaries: `ticker`, `recommendation` (from judge_recommendation), `conviction` (from judge_conviction), `margin_of_safety` (from judge_margin), `position_size` (post-scaling), `veto_score` (from strategist_veto).
7. Return dict:
```python
{
    "positions": positions,
    "sector_allocations": sector_allocations,
    "total_allocation": total_allocation,
    "ticker_details": ticker_details,
    "ticker_count": len(ticker_states),
    "failed_tickers": [t for t in all_tickers if t not in positions],
}
```

**Function 6: `run_portfolio(tickers: list[str], max_concurrent: int = 2) -> dict[str, Any]`**
Synchronous convenience wrapper for CLI/script usage:
```python
def run_portfolio(tickers: list[str], max_concurrent: int = 2) -> dict[str, Any]:
    """Synchronous entry point for multi-ticker analysis. Use from CLI/scripts."""
    return asyncio.run(analyze_portfolio(tickers, max_concurrent))
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Ticker input -> orchestrator | User provides ticker symbols. Passed to yfinance which handles invalid tickers gracefully (returns None/empty). |
| LLM API responses | Gemini API responses parsed by individual agents (Phase 3). Orchestrator does not parse LLM output directly. |
| Cross-ticker state | asyncio.to_thread isolates each ticker in its own thread with independent graph instance. No shared mutable state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Denial of Service | Gemini free tier exhaustion | mitigate | asyncio.Semaphore(2) limits concurrent calls. tenacity exponential backoff on 429 errors. ~41 calls per 5-ticker portfolio well within 500 RPD. |
| T-04-05 | Tampering | State leakage between parallel tickers | mitigate | Each ticker gets a fresh graph instance via create_graph() + fresh InvestmentState via create_initial_state(). Only shared data is macro_context (read-only dict). asyncio.to_thread provides thread isolation. |
| T-04-06 | Tampering | Ticker symbol injection | accept | Ticker symbols are strings passed to yfinance API. yfinance returns empty data for invalid symbols; agents handle DataUnavailableError. No SQL or command injection vector -- tickers never used in string interpolation for commands. |
| T-04-07 | Elevation of Privilege | portfolio_optimize sector scaling | mitigate | Sector cap enforcement uses multiplicative scaling (never increases position size). All position sizes capped at MAX_POSITION_SIZE before sector scaling. Total allocation capped at sum of individual caps. |
</threat_model>
