  <action>
  Create src/lockin/agents/optimizer.py:

  ```python
  # Constants per Notion spec
  MAX_POSITION_SIZE = 0.10          # 10% max per position (not 12%)
  MAX_SECTOR_ALLOCATION = 0.325     # 30-35% max per sector (use midpoint)
  KELLY_FRACTION = 0.33             # Kelly/3 for safety (not Kelly/4)
  CIRCUIT_BREAKER_OVERRIDE_CAP = 0.02  # 1-2% max when circuit breaker overridden
  ```

  ```python
  def kelly_criterion(win_prob: float, win_loss_ratio: float) -> float:
  ```
  - f* = (p * b - q) / b where p=win_prob, b=win_loss_ratio, q=1-p
  - Return max(0, f*) -- never negative (don't short in v1)

  ```python
  def optimizer(state: InvestmentState, config: RunnableConfig) -> dict:
  ```

  Implementation:
  1. Read from state, preferring JudgeOutput if available:
     - ticker = state["asset_ticker"]
     - judge_output = state.get("judge_output")  # JudgeOutput dataclass from judge_math
     - If judge_output available, use its fields directly:
       - recommendation = judge_output.recommendation  # BUY | HOLD | PASS
       - kelly_fraction = judge_output.kelly_fraction   # Already Kelly/3 from judge_math
       - p_success = judge_output.p_success
       - precio_target = judge_output.precio_target
       - valor_mediano = judge_output.valor_mediano
       - circuit_breaker = judge_output.circuit_breaker
       - circuit_breaker_override = judge_output.circuit_breaker_override
     - Fallback to individual state fields if judge_output not available

  2. If recommendation is "PASS" or circuit_breaker is True (without override):
     - position_size = 0.0 (no allocation)
     - Note: recommendation is BUY/HOLD/PASS (not SELL -- spec does not have SELL)

  3. If recommendation is "HOLD":
     - position_size = 0.0 (no new allocation, hold existing if any)

  4. If recommendation is "BUY":
     - The kelly_fraction from JudgeOutput is already Kelly/3 (computed in judge_math)
     - position_size = kelly_fraction
     - Apply per-position cap: position_size = min(position_size, MAX_POSITION_SIZE)

  5. Circuit breaker override path:
     - If circuit_breaker_override is True:
       - Hard cap: position_size = min(position_size, CIRCUIT_BREAKER_OVERRIDE_CAP)
       - This handles the HITL case where user overrode a circuit breaker

  6. Sector allocation (simplified for single-asset v1):
     - Determine sector from yfinance: yf.Ticker(ticker).info.get("sector", "Unknown")
     - Apply MAX_SECTOR_ALLOCATION cap
     - optimizer_sectors = {sector: position_size}

  7. Portfolio metrics:
     - Get current_price from yfinance
     - expected_return = position_size * (valor_mediano / current_price - 1) if valor_mediano > 0
     - portfolio_risk = position_size * consensus_sigma if available
     - sharpe = expected_return / portfolio_risk if portfolio_risk > 0 else 0

  8. Invoke LLM (MODEL_FLASH) for narrative:
     - System: "You are a portfolio optimizer. Explain the position sizing rationale using Kelly Criterion and risk constraints."
     - Include: Kelly/3 fraction, caps applied, final size, risk metrics, circuit breaker status.

  9. Return:
     ```python
     return {
         "optimizer_portfolio": {ticker: position_size},
         "optimizer_sectors": {sector: position_size},
         "optimizer_rebalancing": [],  # v1: no existing positions
         "optimizer_metrics": {
             "kelly_fraction": kelly_fraction,  # Already Kelly/3
             "position_size": position_size,
             "position_cap_applied": position_size == MAX_POSITION_SIZE,
             "circuit_breaker_override_applied": circuit_breaker_override,
             "expected_return": expected_return,
             "portfolio_risk": portfolio_risk,
             "sharpe": sharpe,
             "max_drawdown_estimate": -portfolio_risk * 2,
         },
         "optimizer_narrative": narrative,
     }
     ```
  </action>

  </verify>
