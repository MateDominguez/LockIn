  ```python
  def strategist(state: InvestmentState, config: RunnableConfig) -> dict:
  ```

  Implementation:
  1. Get ticker from state["asset_ticker"]

  2. Fetch earnings transcript from FMP API (if fmp_api_key configured):
     - URL: `https://financialmodelingprep.com/api/v3/earning_call_transcript/{ticker}?apikey={key}`
     - Use httpx.get() with 10s timeout
     - Cache response (simple module-level dict cache by ticker)
     - If FMP key not set or request fails, skip transcript analysis gracefully

  3. Fetch analyst consensus from yfinance:
     - `yf.Ticker(ticker).recommendations_summary` for recent analyst ratings
     - Extract: buy/hold/sell counts, target price consensus
     - If unavailable, proceed without

  4. Build LLM prompt (MODEL_FLASH) for sentiment analysis:
     - System: "You are a financial strategist analyzing earnings calls and analyst consensus. Extract: 1) Overall sentiment (bullish/neutral/bearish, 0-1 score), 2) Key topics/keywords from transcript, 3) Management tone assessment, 4) VeTO score (0-1, organizational health indicator based on language patterns). Respond in JSON."
     - Human: Include transcript excerpt (first 3000 chars to stay within context), analyst summary, ticker info.

  5. Parse LLM response, extract sentiment score and VeTO score.

  6. Build strategic_signals dict:
     ```python
     {
         "earnings_sentiment": sentiment_label,    # "bullish"/"neutral"/"bearish"
         "analyst_consensus": consensus_summary,    # from yfinance
         "transcript_keywords": keywords,           # from LLM extraction
         "veto_score": veto_score,                  # 0-1 informational only
     }
     ```

  7. Return InvestmentState-compatible dict:
     ```python
     return {
         "strategist_veto": veto_score,      # 0-1, informational only
         "strategist_sentiment": sentiment,   # 0-1
         "strategic_signals": signals_dict,
         "strategist_narrative": narrative,    # LLM-generated
         "strategist_confidence": confidence,  # 0-1
     }
     ```

  Important: VeTO is INFORMATIONAL ONLY -- does not trigger veto. CONTEXT.md is explicit about this.
