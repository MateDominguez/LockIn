  ```python
  def bear(state: InvestmentState, config: RunnableConfig) -> dict:
  ```

  CRITICAL DESIGN DECISION (from CONTEXT.md): Bear receives ticker + raw data, NOT Bull's output. Bear builds its thesis completely blind to Bull's analysis.

  Implementation:
  1. Get ticker from state["asset_ticker"]
  2. Fetch fundamentals independently: `data = get_fundamentals(ticker, store=False)`
  3. Increment bull_iteration: `current_iteration = state.get("bull_iteration", 0)`

  4. Compute deterministic red flag signals:
     - Revenue decline: compare total_revenue to prior year if available
     - Margin compression: operating_income/total_revenue trend
     - Debt escalation: total_debt/total_equity ratio > 2.0
     - Cash burn: free_cash_flow negative
     - Earnings quality: abs(net_income - free_cash_flow) / abs(net_income) > 0.5 (accrual gap)

  5. Build pessimistic valuation:
     - Use EPV with pessimistic assumptions: WACC=0.12 (higher risk premium), tax_rate=0.25
     - Apply 20% haircut to EBIT for margin compression scenario
     - Generate distribution: mean = pessimistic_value, std = 25% of mean (wider uncertainty)
     - Compute P10, P25, P50, P75, P90 via scipy.stats.norm

  6. Invoke LLM (MODEL_PRO) for bearish thesis:
     - System: "You are a short-seller and forensic accountant. Your job is to find every reason NOT to invest. Analyze financial data for: 1) Revenue sustainability risks, 2) Margin compression threats, 3) Competitive moat deterioration, 4) Accounting red flags (accrual gaps, inventory build-up), 5) Macro headwinds. Be specific with numbers. Respond with JSON: {challenges: [...], thesis: '...', red_flags: [...], conviction: 0.0-1.0}"
     - Human: Include raw fundamentals data, computed red flags, macro context from state if available.

  7. Return:
     ```python
     return {
         "bear_challenges": challenges_list,
         "bear_valuation_distribution": distribution_dict,
         "bear_thesis": thesis_string,
         "bear_red_flags": red_flags_list,
     ```
