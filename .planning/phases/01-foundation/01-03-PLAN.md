**Part A: Modify judge node in builder.py for HITL interrupt**

Create a new judge wrapper function that adds HITL logic. The key pattern from research: `interrupt()` causes the node to pause. When resumed with `Command(resume=value)`, the node RE-EXECUTES from the beginning, and `interrupt()` returns the resume value instead of pausing.

Modify `src/lockin/graph/builder.py`:

1. Import `interrupt` and `Command` from `langgraph.types`.

2. Create a `judge_with_hitl` function that wraps mock_judge:
```python
from langgraph.types import interrupt

JUDGE_HITL_THRESHOLD = 0.5  # Conviction below this triggers HITL

def judge_with_hitl(state: dict, config: RunnableConfig) -> dict:
    """Judge node with HITL interrupt for low-conviction decisions.

    CRITICAL: Do NOT place side effects before interrupt().
    This entire node re-executes when resumed.
    """
    from lockin.agents.mock import mock_judge

    # First, run the judge logic to get the recommendation
    result = mock_judge(state, config)

    # Check if HITL is needed
    conviction = result.get("judge_conviction", 1.0)
    if conviction < JUDGE_HITL_THRESHOLD:
        # Pause for human review
        human_input = interrupt({
            "reason": "Low conviction -- human review required",
            "conviction": conviction,
            "recommendation": result.get("judge_recommendation"),
            "narrative": result.get("judge_narrative"),
        })
        # After resume, human_input contains the Command(resume=...) value
        result["human_review"] = human_input
        result["judge_hitl"] = True
        result["judge_hitl_reason"] = f"Conviction {conviction} below threshold {JUDGE_HITL_THRESHOLD}"
    else:
        result["judge_hitl"] = False
        result["judge_hitl_reason"] = ""

    return result
```
