  <action>
    Create `src/lockin/data/validator.py` with class `DataValidator`:

    **Constructor:**
    - `__init__(self, database_url: str = "")` — stores database_url for audit logging of HITL triggers. If empty, HITL events log to stderr (same fallback as audit.py).

    **Method: `validate_fundamentals(self, current: FundamentalsResult, previous: FundamentalsResult | None = None) -> ValidationResult`**

    Logic:
    1. **Missing fields check:** For each field in `REQUIRED_FUNDAMENTAL_FIELDS`, check if the field is missing from `current` or is None. Collect into `missing_fields` list.
    2. **Quality score:** `quality_score = 1.0 - (len(missing_fields) / len(REQUIRED_FUNDAMENTAL_FIELDS))`. Clamp to [0.0, 1.0].
    3. **Outlier detection (only if `previous` is provided):** For each numeric field present in BOTH current and previous:
       - Skip if either value is None or previous value is 0.
       - Compute `change_pct = abs((current_val - previous_val) / previous_val) * 100`.
       - If `change_pct > 200`: set `outlier_flags[field] = True`, set `hitl_required = True`, set `hitl_reason` to describe the field and change percentage.
         - Log to audit_logs via `log_audit_event(self._database_url, "validation", "data_validator", "hitl_trigger", {"field": field, "change_pct": change_pct, "ticker": current.get("ticker", "unknown")})`.
       - If `50 < change_pct <= 200`: set `outlier_flags[field] = True` (warning level, no HITL).
    4. Return `ValidationResult` with quality_score, missing_fields, outlier_flags, hitl_required (default False), hitl_reason (default "").

    **Static method: `validate_period_change(field: str, current_val: float | None, previous_val: float | None) -> tuple[bool, str]`**
    - Returns `(is_outlier, severity)` where severity is `""`, `"warning"` (50-200%), or `"hitl"` (>200%).
    - Used internally and available for external callers.

    Import `log_audit_event` from `lockin.utils.audit` for HITL trigger logging. Import types from `lockin.data.types`.
  </action>

    **storage.py** — Create `src/lockin/data/storage.py` with two public functions:

    `store_fundamentals(database_url: str, ticker: str, data: FundamentalsResult, fiscal_year_end: date, period: str = "annual") -> None`
    - Opens a NEW short-lived `psycopg.connect(database_url)` connection (same pattern as audit.py — NEVER share with LangGraph checkpointer).
    - For each financial field in the FundamentalsResult (total_revenue, net_income, gross_profit, operating_income, ebitda, diluted_eps, free_cash_flow, total_assets, total_debt, cash_and_equivalents, total_equity):
      - Skip if value is None.
      - Execute UPSERT: `INSERT INTO fundamentals (ticker, fiscal_year_end, period, field_name, value, source, fetched_at, as_of_date) VALUES (%s, %s, %s, %s, %s, %s, %s, %s) ON CONFLICT (ticker, fiscal_year_end, period, field_name) DO UPDATE SET value = EXCLUDED.value, fetched_at = EXCLUDED.fetched_at, as_of_date = EXCLUDED.as_of_date;`
    - Commit transaction after all fields.
    - If database_url is empty, log to stderr and return (same fallback pattern as audit.py).

    `store_macro_data(database_url: str, data: MacroResult) -> None`
    - Opens a NEW short-lived `psycopg.connect(database_url)` connection.
    - For each indicator in FRED_SERIES keys (gdp, cpi, core_pce, fed_funds, yield_10y_2y, yield_10y_3m, unemployment):
      - Skip if value is None in the MacroResult.
      - Execute UPSERT: `INSERT INTO macro_data (indicator, series_id, value, observation_date, source, fetched_at, as_of_date) VALUES (%s, %s, %s, %s, %s, %s, %s) ON CONFLICT (indicator) DO UPDATE SET value = EXCLUDED.value, series_id = EXCLUDED.series_id, observation_date = EXCLUDED.observation_date, fetched_at = EXCLUDED.fetched_at, as_of_date = EXCLUDED.as_of_date;`
    - Commit after all indicators.
    - If database_url is empty, log to stderr and return.

    Define `FRED_SERIES_IDS` mapping at module level (indicator name -> FRED series ID) for the series_id column. Import from fred_source or duplicate the constant (prefer duplication to avoid circular imports — it's a simple dict).

    **setup_data_tables.py** — Create `scripts/setup_data_tables.py`:
    - Follow the exact pattern of the existing `scripts/setup_db.py`.
    - Load .env from project root, get DATABASE_URL.
    - Create `fundamentals` table with schema:
      ```sql
      CREATE TABLE IF NOT EXISTS fundamentals (
          id              BIGSERIAL PRIMARY KEY,
          ticker          TEXT NOT NULL,
          fiscal_year_end DATE NOT NULL,
          period          TEXT NOT NULL,
          field_name      TEXT NOT NULL,
          value           DOUBLE PRECISION,
          source          TEXT NOT NULL DEFAULT 'yfinance',
          fetched_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
          as_of_date      DATE,
          UNIQUE(ticker, fiscal_year_end, period, field_name)
      );
      CREATE INDEX IF NOT EXISTS fundamentals_ticker_idx ON fundamentals(ticker);
      CREATE INDEX IF NOT EXISTS fundamentals_fetched_at_idx ON fundamentals(fetched_at DESC);
      ```
    - Create `macro_data` table with schema:
      ```sql
      CREATE TABLE IF NOT EXISTS macro_data (
          id              BIGSERIAL PRIMARY KEY,
          indicator       TEXT NOT NULL UNIQUE,
          series_id       TEXT NOT NULL,
          value           DOUBLE PRECISION,
          observation_date DATE,
          source          TEXT NOT NULL DEFAULT 'fred',
          fetched_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
          as_of_date      DATE
      );
      CREATE INDEX IF NOT EXISTS macro_data_fetched_at_idx ON macro_data(fetched_at DESC);
      ```
    - Print progress messages matching the existing setup_db.py style.
    - The script is idempotent (CREATE TABLE IF NOT EXISTS, CREATE INDEX IF NOT EXISTS).
