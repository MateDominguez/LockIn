<task type="auto">
  <name>Task 2: Create TTL cache with stale fallback</name>
  <files>
    src/lockin/data/cache.py
  </files>
  <action>
    Create `src/lockin/data/cache.py` with a `TTLCache` class:

    Use a `@dataclass` `CacheEntry` with fields: `data: Any`, `fetched_at: datetime`.

    `TTLCache` class with:
    - `__init__(self)`: initialize `_store: dict[str, CacheEntry] = {}`
    - `get(self, key: str, ttl_seconds: int) -> Any | None`: Return cached data if within TTL, else None.
    - `get_stale(self, key: str) -> Any | None`: Return cached data regardless of TTL (for stale fallback on re-fetch failure). Return None only if key never cached.
    - `set(self, key: str, data: Any) -> None`: Store data with current UTC timestamp.
    - `clear(self) -> None`: Empty the store (for testing).

    TTL check: `(datetime.now(timezone.utc) - entry.fetched_at).total_seconds() > ttl_seconds`.

    Define module-level TTL constants:
    - `TTL_FUNDAMENTALS = 86400` (24 hours in seconds)
    - `TTL_MACRO = 604800` (7 days in seconds)

    The cache is instance-based (not a singleton) so tests can inject a fresh cache. No external dependencies.
  </action>
  <verify>
    Run a quick inline test:
    ```
    cd /home/mateo/dev/LockIn && python -c "
    from lockin.data.cache import TTLCache, TTL_FUNDAMENTALS, TTL_MACRO
    c = TTLCache()
    c.set('test', {'value': 42})
    assert c.get('test', 9999) == {'value': 42}, 'Should return cached'
    assert c.get('test', 0) is None, 'Should expire with 0 TTL'
    assert c.get_stale('test') == {'value': 42}, 'Stale should ignore TTL'
    assert c.get('missing', 9999) is None, 'Missing key returns None'
    c.clear()
    assert c.get_stale('test') is None, 'Clear empties cache'

    "
    ```
